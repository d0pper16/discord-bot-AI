Discord Rich Presence Mode

    

      Pilih cara bot menampilkan info map di profile Discord-nya:
    


    

    
    

    

    

      
      
    


    

      📚 Tutorial setup Rich Presence (Discord Developer Portal)
      

        Langkah-langkah aktifkan Rich Presence dengan image asset:
        
    
          
  1. Buka Discord Developer Portal
    2. 
          
  2. Pilih aplikasi bot kamu (atau klik New Application kalau belum ada)
    3. 
          
  3. Tab General Information → copy Application ID
    4. 
          
  4. SSH ke VPS, edit .env:
            
    nano ~/discord-bot-AI/.env
# tambahkan:
DISCORD_APP_ID=<app_id_yang_kamu_copy>
    
          
    5. 
          
  5. (Opsional) Upload asset image untuk fallback:
            
      
              
    • Tab Rich Presence → Art Assets di Developer Portal
      • 
              
    • Klik Add Image(s)
      • 
              
    • Upload 2 image:
                
        
                  
      • Asset key: roblox-banner (image rectangular 1024×1024 max, banner default)
        • 
                  
      • Asset key: roblox-icon (image square 512×512, icon kecil default)
        • 
                
      
              
      • 
              
    • Save (Discord proses ~10 menit)
      • 
            
    
          
    6. 
          
  6. Restart bot: pm2 restart yanto
    7. 
          
  7. Pilih radio Rich Presence (image asset) di atas → Simpan
    8. 
        


        Yang akan tampil di Discord:
        
┌──────────────────────────────────────┐
│ 🎮 yanto                          APP│
│                                       │
│  ┌────────┐  Watching <Map Name>     │  ← name (besar)
│  │ banner │  1.4K Active | 2.1M Visit │  ← details
│  │ image  │  roblox.com/games/<id>    │  ← state
│  └────────┘                           │
│                                       │
│         ┌─────────────────┐           │
│         │ Play on Roblox  │ ← button  │
│         └─────────────────┘           │
└──────────────────────────────────────┘


        Catatan penting:
        
    
          
  • Bot biasa tidak selalu render rich presence asset penuh — fitur itu native untuk RPC client (mis. Steam, Roblox client). Kalau gagal render, fallback ke "Watching <Map>" biasa.
    • 
          
  • Asset key (roblox-banner / roblox-icon) harus persis sama dengan yang di-upload di Developer Portal.
    • 
          
  • Discord cache asset ~10 menit. Setelah upload, tunggu sebentar baru terlihat efeknya.
    • 
          
  • Untuk dynamic image dari URL (bukan asset key), pakai field largeImageURL — tapi support-nya tergantung versi discord.js.
    • 
        


        Tutorial setup Streaming Mode:
        
    
          
  1. Pilih radio Streaming (link clickable)
    2. 
          
  2. Isi field URL dengan format Twitch/YouTube. Tidak harus ada channel asli — Discord cuma validasi domain pattern. Contoh:
            
      
              
    • https://www.twitch.tv/yantobot (default kalau dikosongin)
      • 
              
    • https://www.youtube.com/@channelkamu
      • 
            
    
          
    3. 
          
  3. Save → bot di Discord akan tampil dengan badge merah "LIVE" + judul "<Map> | 1.4K Active..." yang clickable buka URL kamu
    4. 
          
  4. Kalau mau redirect ke roblox.com, kamu bisa setup channel Twitch yg deskripsinya berisi link Roblox kamu, atau pakai mode custom (1 baris emoji tanpa link).
    5. 
        

      

    

  







  
Connection

  

    Channel ID + 2 API Gemini bisa diubah lewat dashboard. Disimpan di data/runtime.json
    (override .env). DISCORD_TOKEN tidak bisa diubah dari sini - tetap di .env
    saat deploy. Kalau token Discord salah/error/kosong, bot mati setelah 3x retry. Kalau API/channel
    salah, bot tetap hidup tapi tidur (tidak respon) sampai diperbaiki di sini.
  


  

    

    

      5 slot Gemini API Key (min 1 wajib, sisanya opsional fallback berurutan).
      Bot pakai API ke-1 dulu, switch ke ke-2 kalau limit, dst.
      Kosongin field = pakai value yang sudah ada.
    


    
    
    
    
    

    


    

      
      
    

    

      Server VALIDASI dulu setiap key yang diisi sebelum save. Kalau valid: bot ucap hello di channel.
      Kalau gagal: pesan error per-field, runtime.json TIDAK diubah.

Ingatan Buatan (Q&A pasangan custom dari dashboard)

  

    Tambahkan pasangan pertanyaan-jawaban yang akan dipakai bot SEBELUM ke cache/Gemini.
    Bot mencocokkan dengan similarity (Jaccard) >= 0.85.
    
Format placeholder yang bisa dipakai di jawaban:
    {nama} -> nama bot.
    <@123456789> -> tag user (pakai Discord User ID).
    <@&987654321> -> tag role (Discord Role ID).
    
Contoh:
    
Q: {nama} siapa pembuatmu?
    
A: pembuatku adalah <@d0pper_user_id>, salah satu admin di server ini
